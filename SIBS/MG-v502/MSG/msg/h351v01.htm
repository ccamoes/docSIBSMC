(1) -  Deve obedecer ao seguinte formato: BBBBAAAANN (‘B’ cód. banco + ‘A’ cód. agência + ‘N’ n.º máq.)
 
(2) -  Na ausência de informação, o campo deve ser preenchido a ‘999’. Neste caso, a data de fecho contabilístico (campo 112) também deve estar preenchida a ‘99999999’ e o código de operador (campo 1002) deve estar preenchido com ‘<’, assim, a SIBS devolve uma lista com os períodos a partir do mais recente.
No caso de preenchimento do campo 118 com número de período local, o campo 112 deve ter um conteúdo coerente com o mesmo, caso contrário, a mensagem é recusada com erro.
 
(3) -  Deve obedecer ao seguinte formato: AAAAMMDD, excepto no caso referido no ponto anterior em que pode ser preenchido com ‘99999999’.
 
(4) - Valores possíveis:   "="  Indica que se pretendem obter os elementos iguais aos dados indicados.
">"  Indica que se pretendem obter os elementos superiores aos dados indicados, i.e., os valores seguintes.
"<"  Indica que se pretendem obter os elementos inferiores a partir dos dados indicados, i.e., os valores anteriores.
 
(5) - Valores possíveis: 

01 a 10

(*)  Campo que faz parte do grupo de ocorrências.
