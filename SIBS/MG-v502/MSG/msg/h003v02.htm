V02 -  Relativamente à V01:
  • Foi incluído um novo campo na resposta, designado de Referência para Resposta, com o código de atributo 1293.
    Este campo é necessário para impressão do comprovativo do carregamento, pois a informação introduzida no campo 250 - Referência de Pagamento, pode diferir da Referência para resposta.
  • Em resultado do ponto anterior, o comprimento máximo da mensagem S003, passou de 95 para 104 bytes.
  • Foi alterado o capital social da Optimus, devendo essa mesma alteração ser reflectida nos casos em que se emite um talão com características de factura/recibo.
  • Foram incluídas duas novas operações - Carregamentos Vodafone e Pagamento Sapo ADSL.
  • No caso das operações Carregamento Netpac e Pagamento Sapo foram alterados os dados para efeitos de impressão de talão com características de factura/recibo.
  • No caso da operação Pagamentos ao Estado, o campo 687 - Tipo Pagamento/Serviço passou a ser preenchido a zeros.
  • Foi descontinuada a operação Pagamentos 95.


Validações adicionais

Efectuadas pela SIBS, específicas de cada serviço:


  • Operação "Carregamento TMN"
    • O campo 468 - Entidade tem de ser igual a ‘10091’.
    • O campo 250 - Referência do Pagamento é preenchido com o N.º de Referência que o cliente indicar.
    • O campo 008 - Montante só pode aceitar ser preenchido com os seguintes valores em euros: 5,00; 10,00; 25,00; 50,00; 75,00; 100,00 e 150,00.
    • O campo 687 - Tipo Pagamento/Serviço é igual a ‘010’.
    • O campo 688 - Refª Serv. Esp. MB, deve ser preenchido a zeros.
    • O campo 689 - Refª Adicional deve ser preenchido a zeros.
    • O campo 205 - N.º de Contribuinte do Titular, deve ser preenchido a zeros, excepto se o cliente solicitar a emissão de um documento comprovativo, para efeitos fiscais. Neste caso deve ser preenchido com um Número de Contribuinte válido.


  • Operação "Carregamento Netpac" 
    • O campo 468 - Entidade tem de ser igual a ‘10089’.
    • O campo 250 - Referência do Pagamento é preenchido com o N.º de Referência que o cliente indicar.
    • O campo 008 - Montante só pode ser preenchido de acordo com as seguintes regras:
Campo 250 Montante válido em euros
2º dígito da Referência (de 9 dígitos) é igual a ‘0’, ‘1’, ‘3’, ‘4’, ‘5’, ‘6’ ou ‘7’. 5,00 e 24,94
2º dígito da Referência (de 9 dígitos) é igual a ‘2.’ 5,00 ;  24,94 ;  49,88
2º dígito da Referência (de 9 dígitos) é igual a ‘8.’ 29,43 ;  79,31 ;  144,15
2º dígito da Referência (de 9 dígitos) é igual a ‘9.’ 10,00 ;  30,00 ;  50,00
    • O campo 687 - Tipo Pagamento/Serviço é igual a ‘015’.
    • O campo 688 - Refª Serv. Esp. MB, deve estar preenchido a zeros.
    • O campo 689 - Refª Adicional deve estar preenchido a zeros.
    • O campo 205 - N.º de Contribuinte do Titular, deve ser preenchido a zeros, excepto se o cliente solicitar a emissão de um documento comprovativo, para efeitos fiscais. Neste caso deve ser preenchido com um Número de Contribuinte válido.


  • Operação "Pagamento Sapo" 
    • O campo 468 - Entidade tem de ser igual a ‘10079’ (carregamento em euros).
    • O campo 250 - Referência do Pagamento é preenchido com o N.º de Referência que o cliente indicar.
    • O campo 008 - Montante tem de ser preenchido com um valor superior a 4,99 Euros.
    • O campo 687 - Tipo Pagamento/Serviço é igual a ‘020’.
    • O campo 688 - Refª Serv. Esp. MB, deve estar preenchido a zeros.
    • O campo 689 - Refª Adicional deve estar preenchido a zeros.
    • O campo 205 - N.º de Contribuinte do Titular, deve ser preenchido a zeros, excepto se o cliente solicitar a emissão de um documento comprovativo, para efeitos fiscais. Neste caso deve ser preenchido com um Número de Contribuinte válido.


  • Operação "Pagamento Sapo ADSL" 
    • O campo 468 - Entidade tem de ser igual a ‘10082’.
    • O campo 250 - Referência do Pagamento é preenchido com o N.º de Referência que o cliente indicar.
    • O campo 008 - Montante tem de ser preenchido com um valor igual a 15,00, 22,50 ou 45,00.
    • O campo 687 - Tipo Pagamento/Serviço é igual a ‘025’.
    • O campo 688 - Refª Serv. Esp. MB, deve estar preenchido a zeros.
    • O campo 689 - Refª Adicional deve estar preenchido a zeros.
    • O campo 205 - N.º de Contribuinte do Titular, deve ser preenchido a zeros, excepto se o cliente solicitar a emissão de um documento comprovativo, para efeitos fiscais. Neste caso deve ser preenchido com um Número de Contribuinte válido.


  • Operação "Pagamentos ao Estado"

São possíveis pagamentos referentes a DUCs (Documento Único de Cobrança) com Emissão Prévia, Auto-Liquidações e ainda os documentos que foram submetidos via Internet.

  • O campo 468 - Entidade é preenchido com ‘10095’.
  • O campo 008 - Montante deve ser preenchido com o valor em euros indicado pelo cliente. No caso de documentos com emissão prévia, esse valor é validado pela SIBS contra o ficheiro de documentos informado pela Direcção Geral do Tesouro.
  • Os campos 205 - N.º de Contribuinte do Titular, 689 - Refª Adicional e 250 - Referência do Pagamento, são preenchidos a zeros.
  • O campo 688 - Refª Serv. Esp. MB  é preenchido com o valor (15 dígitos) introduzido pelo cliente e que corresponde aos elementos constantes na "Referência a Pagamento" presente no documento que tem em sua posse.
  • O campo 687 - Tipo Pagamento/Serviço é preenchido a zeros.


  • Operação "Pagamentos Via Card"
  • O campo 468 - Entidade tem de ser igual a ‘10076’.
  • O campo 250 - Referência do Pagamento é preenchido com o N.º de Referência que o cliente indicar.
  • O campo 008 - Montante só pode estar preenchido com os seguintes valores em euros: 15,00; 25,00 e 40,00.
  • O campo 687 - Tipo Pagamento/Serviço = ‘040’.
  • O campo 688 - Refª Serv. Esp. MB, deve estar preenchido a zeros.
  • O campo 689 - Refª Adicional deve ser preenchido a zeros.
  • O campo 205 - N.º de Contribuinte do Titular, deve ser preenchido a zeros, excepto se o cliente solicitar a emissão de um documento comprovativo, para efeitos fiscais. Neste caso deve ser preenchido com um Número de Contribuinte válido.


  • Operação "Carregamento de Cartões Optimus"
  • O campo 468 - Entidade tem de ser igual a ‘10075’.
  • O campo 250 - Referência do Pagamento é preenchido com o N.º de Referência que o cliente indicar.
  • O campo 008 - Montante tem de ser preenchido com um valor superior a 7,47 Euros.
  • O campo 687 - Tipo Pagamento/Serviço = ‘050’.
  • O campo 688 - Refª Serv. Esp. MB, deve estar preenchido a zeros.
  • O campo 689 - Refª Adicional deve ser preenchido a zeros.


  • Operação "Carregamento de Pacotes de Mensagens Optimus"
  • O campo 468 - Entidade tem de ser igual a ‘10073’.
  • O campo 250 - Referência do Pagamento é preenchido com o N.º de Referência que o cliente indicar.
  • O campo 008 - Montante tem de ser igual a 5,00; 10,00 ou 20,00 Euros.
  • O campo 687 - Tipo Pagamento/Serviço = ‘051’.
  • O campo 688 - Refª Serv. Esp. MB, deve estar preenchido a zeros.
  • O campo 689 - Refª Adicional deve ser preenchido a zeros.


  • Operação "Carregamentos Vodafone"
  • O campo 468 - Entidade tem de ser igual a ‘10074’.
  • O campo 250 - Referência do Pagamento é preenchido com o N.º de Referência que o cliente indicar.
  • O campo 008 - Montante tem de ser preenchido com um valor superior a 4,99 euros.
  • O campo 687 - Tipo Pagamento/Serviço = ‘060’.
  • O campo 688 - Refª Serv. Esp. MB, deve estar preenchido a zeros.
  • O campo 689 - Refª Adicional deve ser preenchido a zeros.
