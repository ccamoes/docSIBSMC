Regras para preenchimento dos pedidos:

No primeiro pedido, os campos 1709, 320, 117, 105 e 1002 são preenchidos a zeros ou espaços.

Na resposta são devolvidos os registos mais recentes, encontrados no sistema da SIBS.

Caso existam mais ocorrências que as enviadas na resposta, o campo 2146 é preenchido com o valor 1, possibilitando novo pedido ao sistema SIBS.

Neste novo pedido, os campos 1709, 320, 117 e 105 são preenchidos com os valores da última ocorrência, enviados na resposta anterior, e o campo 1002 é preenchido com o código "<".


(1) - Valores possíveis:   "="  Indica que se pretendem obter os elementos iguais aos dados indicados.
">"  Indica que se pretendem obter os elementos superiores aos dados indicados, i.e., os valores seguintes.
"<"  Indica que se pretendem obter os elementos inferiores a partir dos dados indicados, i.e., os valores anteriores.
 
(2) - Máximo 15 ocorrências.
 
(3) - Valores possíveis:   L10 -  Portagens
T10 -  Telefones
V10 -  Via Verde
Q10 -  Access

(*)  Campo que faz parte do grupo de ocorrências. A posição indicada é referente à 1ª ocorrência do registo.
