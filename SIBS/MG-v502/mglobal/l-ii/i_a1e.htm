CARACTERIZAÇÃO DO PADRÃO EMV


Para possibilitar a emissão de cartões EMV, é necessária uma parametrização prévia de um ou vários Padrões EMV.

A definição de um novo Padrão EMV ou a alteração de parâmetros de um Padrão EMV já existente é efectuada pelo Emissor através da funcionalidade de Pedido de Caracterização de Emissores disponibilizada no Terminal de Serviços SIBS. Para situações de excepção em que tal não seja possível, o Emissor deve preencher e enviar à SIBS este formulário.

Uma vez que a identificação de um Padrão EMV é atribuída sequencialmente, de forma a garantir a coerência da informação no sistema do Emissor e na SIBS, a inserção ou alteração de parâmetros de um Padrão EMV tem efeito simultâneo nos ambientes de Pré-Produção e Produção.


1 . CÓDIGO DO EMISSOR

Preencher com o código numérico atribuído ao Emissor.


2. APLICAÇÃO EMV

SUF.

A coluna "Suf" diz respeito ao sufixo da aplicação que está a ser parametrizada, quando exista (aplicável apenas para Sistema de Pagamento "Visa").
Este campo permite diferenciar duas aplicações semelhantes (i.e., que possuem o mesmo AID) para, por exemplo, parametrizar duas vertentes distintas de 'Crédito' e 'Débito'.
Todos os Sufixos de AID têm que ser caracterizados individualmente, por este mesmo formulário.

PREFERRED NAME

O "Application Preferred Name" é apresentado nos terminais ao cliente, em substituição do "Application Label", sempre que exista a possibilidade de escolha entre várias aplicações para realização de uma determinada transacção, e sempre que as características do terminal o permitam.

O Emissor pode escolher uma curta mnemónica, com um máximo de 6 caracteres. Estes 6 caracteres, em conjunto com um valor parcial retirado do elemento "Application Label", são utilizados para formar o conteúdo do campo "Application Preferred Name".

O "Application Preferred Name" permite a rápida identificação pelo cliente de duas vertentes diferentes para o mesmo cartão, produto "CREDIT" ou "DEBITO", por exemplo.


Exemplo da relação entre "Application Label" e "Preferred Name":

Application Label Application Preferred Name
VISA
VISA ELECTRON
VISA xxxxxx
VISA ELEC xxxxxx

'xxxxxx’ corresponde ao texto determinado pelo Emissor


3. LINGUAGENS SUPORTADAS PELA APLICAÇÃO EMV

As aplicações EMV nos cartões nacionais têm suporte para duas linguagens, sendo a principal obrigatoriamente o português.

A existência de uma segunda linguagem suportada pela aplicação EMV é opcional. O emissor pode determinar a existência de uma segunda linguagem preenchendo este ponto, utilizando para tal uma representação de 2 caracteres alfabéticos, de acordo com a norma ISO 639.

Para a definição de um padrão associado a uma Aplicação de Autenticação, não é considerada uma segunda linguagem, pelo que o ponto 3 não deve ser preenchido.


NÚMERO DO PADRÃO EMV ATRIBUÍDO

A identificação do Padrão EMV é atribuída automática e sequencialmente pela SIBS no momento de inserção desse mesmo Padrão no sistema.

Valores disponíveis para caracterização pelo Emissor: de 002 a 989
(valor "001" reservado para padrão de caracterização da Aplicação Multibanco; o padrão "001" não é parametrizável por Emissor)
