NOME DO FICHEIRO:   ACTUALIZAÇÃO DE ACORDO MB "FICH" = ETSC

 

OBJECTIVO


Este ficheiro é enviado pelo Banco à SIBS sempre que pretenda efectuar alterações, de forma massiva, às condições contratuais, do acordo MB, definidas para estabelecimentos por si matriculados.

 

ESTRUTURA


O ficheiro é constituído pelos seguintes tipos de registo:

  • o registo de Header do ficheiro "TIPREG=0";
  • registos de detalhe "TIPREG=1";
  • o registo de Trailer "TIPREG=9".


O registo tem um comprimento fixo de 46 bytes.

 

NOTAS


Todos os estabelecimentos incluídos no ficheiro devem ter como ‘proprietário’ (entidade responsável pela matrícula no Sistema Multibanco dos terminais instalados no estabelecimento) o código de comerciante do Banco enviado no header do ficheiro. Caso algum dos estabelecimentos possua, no sistema MB, um proprietário diferente, o registo é recusado.

 
O processamento de cada registo de detalhe, consoante o "INDICADOR DE ACTUALIZAÇÃO" informado, é:

  • ‘1’ - Actualização do tipo de serviço – Apenas se processa o campo 1103 - TIPO DE SERVIÇO. Não se valida nem processa o conteúdo dos restantes campos relacionados com a TSC, qualquer que seja o seu conteúdo.
  • ‘2’ - Actualização de dados da TSC – Apenas se processa os campos relacionados com a TSC, isto é, os campos 1111, 1110, 1109, 1108. Não se valida nem processa o conteúdo do campo 1103. Ainda que o Banco não pretenda alterar alguns dos dados relacionados com a TSC, deve enviar valores correctos (isto é, os existentes e a manter) para todos os 4 campos atrás mencionados.
  • ‘3’ - Actualização de tipo de serviço e dados da TSC – Processa-se e integra-se no sistema MB todos os dados enviados pelo Banco no registo. Ainda que o Banco não pretenda alterar algum dos dados relacionado com a TSC, deve enviar um valor correcto (isto é, o existente e a manter) para todos os 4 campos relacionados com esta tarifa.


Os novos valores só entram em vigor quando, após o processamento do ficheiro pela SIBS, os TPA dos comerciantes/estabelecimentos enviados no ficheiro efectuem uma abertura de período contabilístico.

O processamento do ficheiro não altera a informação relativa ao serviço PMB, caso exista, nos estabelecimentos incluídos no ficheiro.

NOME DO FICHEIRO:   ERROS DO FICHEIRO ETSC "FICH" = EETS

 

OBJECTIVO


Este ficheiro é produzido pela SIBS para o Banco sempre que for processado um ficheiro de actualização de acordos MB (ETSC) recebido do Banco, quer tenham ocorrido erros ou não.

 

ESTRUTURA


O ficheiro é constituído pelos seguintes tipos de registo:

  • o registo de Header do ficheiro "TIPREG=0";
  • o registo de detalhe "TIPREG=1", que identifica o ficheiro original tratado, e indica a situação do seu registo de header;
  • um registo de detalhe "TIPREG=2", por cada registo de detalhe errado, no ficheiro original;
  • o registo de Trailer "TIPREG=9".


O registo tem um comprimento fixo de 76 bytes.

 

EETS - TIPO DE REGISTO = 1 (CARACTERIZAÇÃO DO FICHEIRO DE ORIGEM)

Nome do Campo Comp. Pos. Rep. N.º Campo Obs.
TIPREG 1 1 N 111 ='1'
APLIC 1 2 A 101 ='M'
FICH 4 3 A 102 ='ETSC'
VERFICH 2 7 N 064 ='00'
CODBAN 4 9 N 103  
CPD 1 13 N 104  
IDFICH 11 14 N 061  
HDRSIT 1 25 A 521 (Qualquer valor ¹ de '0' significa a rejeição total do ficheiro do Banco)
IDFICH 11 26 N 061 =zeros (não aplicável)

 

EETS - TIPO DE REGISTO = 2 (REGISTO ERRADO)

Nome do Campo Comp. Pos. Rep. N.º Campo Obs.
TIPREG 1 1 N 111 ='2'
NUMOCORR 2 2 N 428 a)
CODERRTSC 3 4 A 1712 b)
REGISTO ERRADO 46 31 N 905 c)


a)   Indica o número de vezes que ocorre o campo seguinte, para o registo que foi rejeitado, no máximo de 9 erros.
b)   Estes erros implicam apenas a rejeição do registo informado no campo seguinte. Caso o número de erros seja inferior a 9, os restantes bytes, que completam as 9 ocorrências, são preenchidos a espaços.
c)   Apresenta o registo tal como foi recebido no ficheiro de origem e sobre o qual é(são) informado(s) o(s) erro(s).
